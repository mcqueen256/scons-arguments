Class Hierarchy

  • object: The most base type
    • SConsArguments._ArgumentDecl: Declaration of single Argument.
    • SConsArguments._Arguments: Correlates construction variables from SCons environment with command-line variables (variable=value) and command-line options (--option=value).
    • SConsArguments._ArgumentsEnvProxy: Proxy used to get/set construction variables in SCons environment while operating on mapped variable names.
    • SConsArguments._VariablesWrapper
    • SConsArguments._missing: Represents missing argument to function.
    • SConsArguments._notfound: Something that has not been found.
    • SConsArguments._undef: Represents undefined/inexistent variable. This is not the same as None
    • dict: dict() -> new empty dictionary dict(mapping) -> new dictionary initialized from a mapping object's (key, value) pairs dict(iterable) -> new dictionary initialized as if via: d = {} for k, v in iterable: d[k] = v dict(**kwargs) -> new dictionary initialized with the name=value pairs in the keyword argument list.
      • SConsArguments._ArgumentDecls: Dictionary-like object to hold declarations of Arguments.
    • type: type(object) -> the object's type type(name, bases, dict) -> a new type
      • SConsArguments._undef_meta: Meta-class for the _undef class
